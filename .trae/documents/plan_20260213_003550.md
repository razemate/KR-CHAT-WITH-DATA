I have updated the plan to use **Gemini 2.5 Flash** as the default LLM model.

### Refined Architecture: Monorepo Deployment
To simplify deployment and management, I will structure this as a **monorepo** deployed to a **single Vercel project**. Vercel automatically handles both the Python backend (Serverless Functions) and the React frontend (Static Site Generation).

### Tech Stack
*   **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS + Lucide Icons
*   **Backend**: Python 3.12 + FastAPI + Vanna AI
*   **Database**: PostgreSQL (Supabase)
*   **AI**: **Google Gemini 2.5 Flash** (Primary) with OpenRouter Fallback

### File Structure
```
KR-CHAT-WITH-DATA/
├── api/                      # Backend (Serverless Functions)
│   ├── index.py              # Vercel Entry Point
│   ├── app.py                # FastAPI Application
│   ├── vanna_calls.py        # Vanna AI Logic (Gemini 2.5 Flash)
│   └── requirements.txt      # Python Dependencies
├── src/                      # Frontend (React Source)
│   ├── components/           # Chat Components
│   ├── hooks/                # API Hooks
│   ├── App.tsx               # Main Application
│   └── main.tsx              # React Entry Point
├── public/                   # Static Assets
├── package.json              # Frontend Dependencies
├── tsconfig.json             # TypeScript Config
├── vite.config.ts            # Vite Configuration
├── vercel.json               # Deployment Configuration (Routing)
└── README.md                 # Documentation
```

### Implementation Plan
1.  **Cleanup**: Remove all existing files to eliminate conflicts.
2.  **Backend Setup (`api/`)**:
    *   **Dependencies**: `requirements.txt` with `vanna`, `fastapi`, `uvicorn`, `psycopg2-binary`, `google-generativeai`. *Explicitly excluding `vanna[all]` to fix the build error.*
    *   **Core Logic**: `vanna_calls.py` implementing `GeminiLlmService` (**Gemini 2.5 Flash**), `FallbackLLM`, and `PostgresRunner`.
    *   **Server**: `app.py` initializing `VannaFastAPIServer`.
3.  **Frontend Setup (`src/` & Root)**:
    *   **Configuration**: `vite.config.ts` configured to proxy `/api` requests to the backend during development.
    *   **UI**: Modern chat interface with message history, loading states, and Markdown rendering for SQL/Data.
    *   **Dependencies**: `package.json` with `react`, `react-dom`, `lucide-react`, `tailwindcss`, `axios`.
4.  **Deployment Configuration**:
    *   **`vercel.json`**:
        *   Route `/api/*` -> `api/index.py` (Python Serverless Function).
        *   Route `/*` -> Frontend (Static Assets).
    *   **Environment**: Ensure `GEMINI_API_KEY`, `OPENROUTER_API_KEY`, and `SUPABASE_CONNECTION_STRING` are set.
5.  **Execution**:
    *   Generate all files.
    *   Use the automated script to deploy to Vercel.

This unified structure ensures a seamless developer experience and a performant production app without managing multiple deployments.
